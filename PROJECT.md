# Signal Deck — Project Anchor

## Canonical Project

**Project name:** Signal Deck

**Canonical folder:** `~/Desktop/Signal Deck`

This folder is now the **one true active build location** for Signal Deck.

## Rule

All future work on the musician app should reference and build toward this folder as the canonical project home.

Older prototype paths or alternate folders should be treated as:
- deprecated
- reference-only
- not the primary active build target

## Project Goal

Build a functioning local musician app for independent artists.

## Current Direction

Signal Deck should be:
- song-first
- simple
- useful
- local-first
- blank on first open
- generic/reusable for other musicians
- not tied to Zach-specific demo data

## Current Priorities

1. Blank-start workspace flow
2. Local save/load behavior
3. Import/export project files
4. Functional buttons, forms, and pages
5. More complete real workflows
6. Integration of Boxman and Martian as restrained guide layer
7. Keep the app simple and avoid bloat

## Non-Goals

Do not drift into:
- fake enterprise SaaS complexity
- over-theorizing without implementation
- overfilled demo content
- Zach-specific seeded private data as the main product state
